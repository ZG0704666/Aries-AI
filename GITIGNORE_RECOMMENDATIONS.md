# 更新 .gitignore - 敏感文件屏蔽规则

以下是推荐补充的 .gitignore 规则（确保敏感文件不被误提交）：

```
# ============================================================
# 本地环境配置与密钥
# ============================================================
local.properties
.env
.env.local
.env.*.local
.env.*.secret
.env.prod

# ============================================================
# Android 签名密钥文件
# ============================================================
*.jks
*.keystore
*.p12
*.pfx
*.pem
*.key
release.keystore

# ============================================================
# Release/Build 产物（可能包含元数据）
# ============================================================
release/
/release/
/outputs/
*.apk
*.aab
*.aar

# ============================================================
# IDE 与构建缓存（可能暴露密钥）
# ============================================================
.gradle/
build/
.idea/
*.iml
.vscode/
*.swp
*.swo

# ============================================================
# 密钥管理与加密文件
# ============================================================
*.encrypted
*.gpg
secrets.json
credentials.json
private_key*
ssh_keys/
```

【应用方式】

1. 编辑项目根目录的 `.gitignore` 文件
2. 添加上述规则（避免重复）
3. 验证没有敏感文件被跟踪：
   ```bash
   git status
   git ls-files | grep -E "(local\.properties|\.env|\.jks|\.keystore|\.p12)"
   ```
4. 如果已误提交敏感文件：
   ```bash
   # 停止跟踪但保留本地文件
   git rm --cached local.properties
   git commit -m "chore: stop tracking local.properties"
   ```
